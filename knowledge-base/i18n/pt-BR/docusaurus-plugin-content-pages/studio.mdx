      <h4>Tipografia e Engenharia de Fontes</h4>
      <p>O sistema utiliza a família de fontes Nunito (Rounded), gerenciada como um ativo centralizado dentro do workspace <code>@studio</code> para garantir paridade entre todos os pacotes. Os títulos são priorizados em pesos Negrito (Bold), enquanto o texto do corpo segue as especificações de peso Regular.</p>

      <ul>
        <li><strong>Distribuição:</strong> As fontes são empacotadas via <code>@fontsource/nunito</code> e injetadas automaticamente nos estilos globais através do <code>theme.css</code> centralizado.</li>
        <li><strong>Performance:</strong> A implementação utiliza <strong>Fontes Variáveis</strong> (<code>variable.css</code>) para fornecer todo o espectro de peso (200-900) com uma única requisição HTTP, otimizando os Core Web Vitals.</li>
      </ul>

    <TabItem value="penpot" label="Penpot & Aide">
      <h3>Engenharia de Design Self-Hosted</h3>
      <p>O Elo Orgânico utiliza um stack Penpot distribuído integrado com o Aide, um assistente de engenharia nativo de IA. Esta configuração garante soberania absoluta dos dados e sincronização automatizada entre as camadas de design e a base de código.</p>

      <h4>Arquitetura de Infraestrutura</h4>
      <p>O ambiente é orquestrado via um stack Docker multi-container, utilizando backends baseados em nuvem para durabilidade.</p>

      - **Orquestração:** Gerenciada via <code>studio/penpot/compose.yaml</code> (Frontend, Backend, Exporter, Valkey).
      - **Camada de Banco de Dados:** Persistência PostgreSQL hospedada no Neon.tech para integridade transacional serverless.
      - **Armazenamento de Objetos:** Ativos binários são persistidos em buckets compatíveis com S3 (Backblaze B2 ou Filebase), garantindo um footprint mínimo no Git.

      <h4>Aide: Integração Nativa de IA</h4>
      <p>O Aide funciona como uma ponte Model Context Protocol (MCP), permitindo que agentes de IA interajam diretamente com a API de Plugin do Penpot para consultas avançadas e transformações de layout.</p>
      
      <h5>Lógica da Ponte Aide</h5>

      - **Protocolo:** Comunicação WebSocket bidirecional entre o servidor <code>@penpot/mcp</code> e o ambiente Penpot.
      - **Segurança:** O Aide opera dentro da rede isolada <code>penpot-net</code>, fornecendo um gateway controlado para manipulação de design.
      - **Conectividade:** Utiliza a Porta 4402 para auditorias de design e sincronização em tempo real.

      <h5>Regras de Manipulação Obrigatórias para IA</h5>

      - **Reutilização de Componentes:** Sempre verifique a biblioteca de componentes existente (Penpot) ou a biblioteca de UI (<code>instance/apps/web/src/components</code>) antes de criar novos elementos. O Aide deve priorizar tokens e componentes existentes do design system.
      - **Lógica Token-First:** Exportações de design para código devem utilizar estritamente Propriedades Personalizadas CSS derivadas de <code>src/tokens/</code> para cores, fontes e espaçamento.
      - **Otimização de SVG:** Garanta que o código SVG exportado seja otimizado, siga os padrões de ícones do projeto e seja integrado via o registro <code>@elo-organico/studio/icons</code>.
      - **Hierarquia de Camadas:** A manutenção do agrupamento semântico é obrigatória. Os elementos devem ser organizados em páginas e quadros claros (ex: <code>Layout/Grid</code>, <code>Components/Buttons</code>).

    <TabItem value="creative-hub" label="Studio & Ativos Criativos">
      <h3>Repositório Criativo Centralizado</h3>
      <p>O workspace Studio atua como o cofre mestre para todos os ativos criativos da marca Elo Orgânico. Ele centraliza arquivos mestres de suítes profissionais de design e produção, garantindo uma evolução visual com controle de versão e eliminando a gestão fragmentada de ativos.</p>

      <h4>Integração de Ferramentas de Design</h4>
      <p>O repositório suporta fluxos de trabalho multiferramentas mantendo arquivos de origem brutos que servem como a origem canônica para todos os ativos prontos para produção.</p>

      - **Gráficos Vetoriais:** Arquivos mestres do Adobe Illustrator (<code>.ai</code>) para logotipos, exploração tipográfica e iconografia.
      - **Imagens Digitais:** Fontes do Adobe Photoshop (<code>.psd</code>) para ativos raster de alta fidelidade e material de marketing.
      - **Engenharia 3D:** Arquivos mestres do Blender (<code>.blend</code>) para componentes da plataforma e visualizações espaciais (ex: <code>Avokado.blend</code>).
      - **Pós-Produção:** Projetos do Adobe Premiere (<code>.prproj</code>) para tutoriais comunitários e conteúdo de marketing.

      <h4>Gestão de Ativos de Origem</h4>
      <p>Todos os arquivos mestres de alta fidelidade são estritamente mantidos dentro da estrutura de diretórios <code>assets/sources/**</code>. Isso garante que toda a história da identidade visual da marca seja preservada e acessível dentro do contexto do monorepo.</p>

      <h4>Pipeline de Produção</h4>
      <p>Os arquivos mestres transitam para as camadas da aplicação através de um pipeline otimizado:</p>

      - **Ícones:** As fontes são processadas em SVGs otimizados em <code>src/icons/</code>.
      - **Modelos 3D:** Arquivos mestres do Blender são exportados para o formato GLB para renderização em runtime.
      - **Tokens:** Constantes visuais são exportadas como definições TypeScript fortemente tipadas de <code>src/tokens/</code>.

      <h4>Exportações de Workspace com Namespace</h4>
      <p>Pacotes internos acessam ativos criativos através de exportações estritamente definidas para garantir o isolamento arquitetural:</p>

      <ul>
        <li><code>@elo-organico/studio/icons</code>: Biblioteca SVG otimizada e registro de componentes Icon curado (wrapper forwardRef).</li>
        <li><code>@elo-organico/studio/logos</code>: Logotipos canônicos da marca.</li>
        <li><code>@elo-organico/studio/tokens</code>: Constantes de design fortemente tipadas para lógica JS/TS.</li>
        <li><code>@elo-organico/studio/theme.css</code>: Tema Tailwind CSS v4 com zero runtime e resets globais.</li>
      </ul>
